- [ ] **Step 1: Create `StoryRightPanel.tsx`**

Create `src/screens/hud/StoryRightPanel.tsx` with this exact content:

```tsx
import type { Actor } from '@nannymud/shared/simulation/types';
import { theme } from '../../ui';
import { AbilityStrip } from './AbilityStrip';

interface Props {
  actor: Actor;
  simTimeMs: number;
  bossSpawned: boolean;
}

const SLOT_COUNT = 6;

export function StoryRightPanel({ actor, simTimeMs, bossSpawned }: Props) {
  return (
    <div style={{ display: 'flex', flexDirection: 'column', alignItems: 'flex-start', gap: 4 }}>
      <div
        style={{
          fontFamily: theme.fontMono,
          fontSize: 10,
          letterSpacing: 3,
          color: bossSpawned ? theme.bad : theme.inkDim,
        }}
      >
        {bossSpawned ? 'P1 · ABILITIES' : 'STANDBY'}
      </div>
      {bossSpawned ? (
        <AbilityStrip actor={actor} side="p1" showKeys={false} simTimeMs={simTimeMs} />
      ) : (
        <div style={{ display: 'flex', gap: 4 }}>
          {Array.from({ length: SLOT_COUNT }).map((_, i) => (
            <div
              key={i}
              style={{
                width: 68,
                height: 92,
                background: theme.panel,
                border: `1px solid ${theme.lineSoft}`,
                borderRadius: 4,
                opacity: 0.3,
              }}
            />
          ))}
        </div>
      )}
    </div>
  );
}
```

## Task 3: Add Escape as secondary pause key

**Root cause:** Escape was intentionally migrated away from being the pause key (because fullscreen hijacks it), but the user wants it as a fallback trigger in addition to P. Hardcoding a secondary check in the adapter avoids touching the bindings system.

Note: in fullscreen mode the browser will exit fullscreen on Escape first; the subsequent Escape event will then pause the game. This two-step is expected behavior.

**Files:**
- Modify: `src/game/input/PhaserInputAdapter.ts:118`

- [ ] **Step 1: Update `pauseJustPressed` in `getInputState`**

In `PhaserInputAdapter.ts`, line 118 currently reads:
```typescript
pauseJustPressed: this.justPressed.has(b.pause),
```

Change it to:
```typescript
pauseJustPressed: this.justPressed.has(b.pause) || this.justPressed.has('Escape'),
```

The adapter's `onKeyDown` handler already adds every key to `justPressed`, so no other changes are needed. Escape will be tracked automatically.
