## vsAI improvements (packages/shared/src/simulation/vsAI.ts)

Complete rewrite of `synthesizeVsCpuInput` to actually use guild strategy fields that were previously ignored:

- **preferRange** → ideal engagement distance (close 55u / mid 180u / long 320u); ranged guilds maintain distance rather than always closing to melee
- **retreatBelowHpPct** → flee when HP falls below configured threshold
- **aggressionPct** → gates whether the bot presses attack each decision tick (not movement speed, as an earlier version wrongly did)
- **useAtCloseRange / useAtLongRange** → ability slot filtering in pickAbilitySlot
- **blockOnReaction** → probability of raising block when player is swinging nearby
- **Heal priority** → if a heal ability is ready and HP < 60%, fires it before considering attacks
- **Jump** → ranged guilds jump while kite-retreating; melee guilds jump to close large gaps
- **Ranged basic attacks** → guilds with rangedBasic now fire it every attack cycle (previously blocked by `!isRanged` guard)
- **Projectile ability weighting** → ranged guilds prefer projectile abilities at distance (+3 weight bonus)

## Known remaining issues

- Cultist vs Mage: 80% — Cultist still hard-counters Mage despite open_the_gate nerf. Psychic DoTs bypass Mage's magic resist offset.
- Viking vs Knight: 75% — Viking's Undying Rage (3s invincibility) remains a strong equaliser against Knight's armor.
- Druid 1v1 weakness is structural: all abilities are heals/CC, zero burst damage. Converges to 42% through sheer sustain.
