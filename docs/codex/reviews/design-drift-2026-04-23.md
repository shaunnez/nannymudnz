# Design Drift Review — 2026-04-23

## Summary

Static comparison of the 16 manifest screens against the `design_handoff_nannymud/` reference package. Coverage is broad: 14 of 16 screen_ids have an implementation mounted through `src/App.tsx`; `mp_results` is an explicit stub, and the `mp_hub` implementation is intentionally scoped down to a name-gated host/join launcher with a "public rooms coming soon" placeholder.

Dominant drift patterns: (1) the MP stack has been re-scoped to strict 1v1 around a Colyseus authoritative server — lobby, char-select, stage-select, and loading are all 2-player/host-authoritative variants of the handoff's 8-player flows (acceptable per the runbook's gameplay-readability exception); (2) the SP battle HUD is a working Phaser-backed overlay (`HudOverlay`) rather than the prototype's fake-sim React HUD — presence of genuine combat log, round timer, ability strip all verified; (3) the prototype's "Team Config" screen (screens-02.jsx `TeamConfig`) has no implementation counterpart, which is fine because the target modes don't exist yet. The highest-signal real gaps are a missing `mp_results` scoreboard, a bare-bones `mp_hub` lacking the table/filter/listed-rooms browser, and an `MpLobby` that ships without a chat panel, room metadata strip, slot team-cycle, or launch-ready room-meta chips.

## Follow-up summary (highest-priority gaps)

- `mp_results` is a stub — build a real 1v1 results screen with winner banner, round summary, and REMATCH / LEAVE CTAs (reuse SP `ResultsScreen` layout bound to server telemetry).
- `mp_hub` public-rooms browser is absent — port the reference `HubTable` (search, mode filter, region/ping/state columns, row-click enter) behind a matchmaking backend when it lands.
- `mp_lobby` is missing the room-meta chip strip (mode/BO#/stage/visibility/FF/spec) and the right-side chat panel; add both to bring parity with handoff screen 16.
- `stage_select` lost the large hero-preview + scrollable list layout — restore it so stage flavor, hue swatch, and pagination dots read clearly (currently a 3×3 grid).
- `settings` is missing audio mix (master/music/sfx), graphics quality chips, and the feedback toggles (screen shake, controller rumble, show ping, combat-log overlay) — add a second column to match reference screen 12.
- `charselect` dropped the hover-preview stat-bar strip under the grid — reintroduce if quick-compare stat density is needed during selection.
- `mp_lobby` lacks a dedicated `[ CHANGE GUILD ]` affordance — wire it to re-enter `mp_cs` with a returnTo target if character swaps before launch become common.

## Preserved divergences (do not regress)

- `game_story` (`src/screens/GameScreen.tsx` + `src/screens/hud/HudOverlay.tsx`): real Phaser-backed HUD replaces the prototype's fake-sim React HUD. Verdict `better`.
- `mp_battle` (`src/screens/mp/MpBattle.tsx`): bridges Colyseus state into the shared `GameScreen` + `HudOverlay` rather than building a separate MP-only HUD. Verdict `better`.
- `menu` (`src/screens/MainMenu.tsx`): unbuilt game modes dimmed with `— SOON` badge. Honest signal to users; keep.
- `guild_dossier` (`src/screens/GuildDossier.tsx`): `SpriteStrip` adds animated guild preview to the hero column; retain.
- `mp_cs` / `mp_load`: 1v1-scoped layouts that reuse SP surfaces. Correct for current scope; do not regress to 8P reference before 8P is actually in scope.
